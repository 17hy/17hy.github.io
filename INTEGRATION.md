# 🔗 主页与私有资料仓库集成指南

## 📌 架构说明

你的GitHub主页系统包含两个仓库：

```
┌─────────────────────────────┐
│  17hy.github.io             │  ← 公开主页
│  ✅ 公开 | 🚀 自动部署      │
│  技术栈：React + Vite       │
└──────────────┬──────────────┘
               │
               │ 链接到
               │
┌──────────────v──────────────┐
│  portfolio-private          │  ← 私有资料
│  🔒 私有 | 📚 详细资料      │
│  内容：文档、源码、案例      │
└─────────────────────────────┘
```

## 🔄 集成流程

### 1️⃣ 主页展示项目列表

在 `portfolio.json` 中定义项目：

```json
"projects": [
  {
    "id": "project-1",
    "name": "项目名称",
    "github": "https://github.com/17hy/project-1",  // 公开仓库
    "demo": "https://demo.com",                      // 在线演示
    "featured": true
  }
]
```

### 2️⃣ 项目卡片上显示链接

主页自动渲染：
- 📖 **演示链接** - 点击查看在线项目
- 🔗 **GitHub链接** - 跳转到公开源代码
- 📁 **资料访问** - 申请访问私有资料库

### 3️⃣ 申请访问流程

```
用户访问主页
    ↓
点击 "查看项目资料" 按钮
    ↓
选择方式 A、B 或 C
    ↓
获得访问权限
    ↓
浏览私有仓库内容
```

## 🌐 三种链接方式

### 方式 A：直接链接（最简单）

```json
"privateResources": {
  "title": "项目资料库",
  "link": "https://github.com/17hy/portfolio-private"
}
```

**优点**：简单直接
**缺点**：访问者看到GitHub私有仓库提示
**适用**：不介意暴露仓库名的场景

### 方式 B：申请表单（推荐）

1️⃣ 创建申请表单（选择一个）：

**Google Forms**
```
https://forms.google.com/
```

**Typeform**
```
https://www.typeform.com/
```

2️⃣ 获得表单嵌入代码

3️⃣ 在主页中配置：

```json
"privateResources": {
  "applyLink": "https://forms.gg/your-form-id",
  "applyCTA": "申请访问"
}
```

**优点**：
- 可以筛选和评估访问申请
- 收集联系信息
- 专业形象

### 方式 C：邮件联系（最灵活）

```json
"privateResources": {
  "contactEmail": "your.email@example.com",
  "applyCTA": "发邮件申请"
}
```

邮件模板示例：

```
主题：Portfolio Access Request

亲爱的XXX：

您好！我对您的项目经历和技术方案很感兴趣，
希望能获得您私有项目资料库的访问权限。

我的背景：[简要介绍]
我的GitHub：[你的GitHub]

感谢！

此致
敬礼
```

## 🔐 邀请协作者

将特定用户添加到私有仓库：

1. 进入 [repository settings](https://github.com/17hy/portfolio-private/settings/access)
2. "Collaborators and teams" → "Add people"
3. 输入GitHub用户名
4. 选择权限级别（通常选Read）

## 📈 跟踪访问数据

### 方式A（直接链接）
使用GitHub的流量统计：
- Settings → Code and automation → Traffic

### 方式B（申请表单）
在表单工具中查看提交数据

### 方式C（邮件链接）
Gmail标签和筛选器

## 🎯 最佳实践

### ✅ 私有仓库应包含

- 📖 项目的详细设计文档
- 💻 完整的源代码
- 📊 性能测试数据和报告
- 🎬 演示视频或截图
- 📝 内部技术分享
- 🏆 项目成果证明（邮件、反馈等）

### ❌ 不应公开

- 商业机密或敏感数据
- 个人隐私信息
- 未经许可的第三方内容
- API密钥或凭证

### 🔄 定期更新

- 每完成新项目，添加到主页
- 更新项目资料库中的详细信息
- 收集反馈和推荐
